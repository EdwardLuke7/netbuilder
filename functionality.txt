FUNCTIONAL REQUIREMENTS

1. Users
	1.1. Secure logins using username and password matching
	1.2. Unique user per employee
	1.3. Must be able to change stock levels

2. Orders
	2.1. Must be able to mark current stock orders as 'checked out' by users
	2.2. Must be printable

3. Database
	3.1. Must store products, stock levels and warehouse locations

4. Products
	4.1. Must have an associated stock level
	4.2. Must have methods for changing stock levels
	4.3. Must have an assocaited location in the warehouse
	4.4. Must have a pourous wear boolean

NON-FUNCTIONAL REQUIREMENTS

1. Logins should be fast (<5 seconds)
2. Orders should list products in nearest-first order
